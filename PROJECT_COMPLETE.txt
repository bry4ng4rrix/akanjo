🔑 LES 3 VARIABLES D'ENVIRONNEMENT
═════════════════════════════════════════════════════════════════════════════════

CRÉEZ UN FICHIER .env.local AVEC CES 3 VARIABLES:

NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_anon_xxxxxxxxxxxx
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxxx

💡 Comment obtenir ces variables?
   • Allez sur supabase.com
   • Ouvrez votre projet
   • Settings → API
   • Copier les 3 valeurs


⚡ DÉMARRAGE EN 5 MINUTES
═════════════════════════════════════════════════════════════════════════════════

🆘 AIDE RAPIDE
═════════════════════════════════════════════════════════════════════════════════

"Cannot connect to Supabase"
→ Vérifiez NEXT_PUBLIC_SUPABASE_URL

"Migrations failed"
→ Vérifiez SUPABASE_SERVICE_ROLE_KEY

"Login échoue"
→ Créez un utilisateur dans Supabase Auth

"Port 3000 déjà utilisé"
→ Utilisez: pnpm dev -- -p 3001

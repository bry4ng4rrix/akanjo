⚡ DÉMARRAGE EN 3 ÉTAPES (5 minutes)

1️⃣  CRÉER UN PROJET SUPABASE GRATUIT
   └─ Aller sur: https://app.supabase.com
   └─ Créer un nouveau projet (attendre 5-10 min)

2️⃣  COPIER LES VARIABLES D'ENVIRONNEMENT
   └─ Dans Supabase → Settings → API
   └─ Copier ces 3 valeurs:
      • Project URL → NEXT_PUBLIC_SUPABASE_URL
      • Anon key → NEXT_PUBLIC_SUPABASE_ANON_KEY  
      • Service Role Key → SUPABASE_SERVICE_ROLE_KEY
   
   └─ Créer un fichier ".env.local" à la racine du projet
   └─ Coller vos valeurs dedans

3️⃣  INSTALLER ET LANCER
   ```bash
   pnpm install
   pnpm db:migrate
   pnpm dev
   ```
   
   └─ Ouvrir: http://localhost:3000

🔑 VARIABLES D'ENVIRONNEMENT NÉCESSAIRES

CRÉER UN FICHIER: .env.local

CONTENU:
───────────────────────────────────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_anon_xxxxxxxxxxxxx
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
───────────────────────────────────────────────────────────────────────

⚠️  IMPORTANT:
- Ne JAMAIS committer .env.local en git
- Garder le SERVICE_ROLE_KEY secret!
- Les autres variables sont publiques (utilisées côté client)

🛠️  COMMANDES UTILES

Développement:
   pnpm dev                    # Démarrer le serveur de dev
   pnpm build                  # Construire pour production
   pnpm start                  # Lancer la version production

Base de données:
   pnpm db:migrate             # Appliquer les migrations
   pnpm db:migrate:status      # Voir le statut des migrations
   pnpm db:seed                # Charger les données d'exemple
   pnpm db:reset               # Réinitialiser complètement

Qualité du code:
   pnpm lint                   # Vérifier le code
